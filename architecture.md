# architecture.md — 単価計算アプリ

## 1. 目的

買い物中に、複数商品の「単価」を同一単位で比較し、最安（または同率最安）を即座に判断できるようにする。

---

## 2. スコープ（MVP）

- 比較対象：初期2件 → 「追加」で最大4件
- 共通単位：g / ml / 個 / 袋（比較単位は全商品共通）
- 単価表示：
  - g → 円/100g
  - ml → 円/100ml
  - 個 → 円/1個
  - 袋 → 円/1袋
  - 表示は小数第1位まで四捨五入
- 最安表示：
  - 単独最安：背景色 + ラベル「こっちがお得！」
  - 同率最安：別背景色 + ラベル「同じ」（同率最安の全商品に表示）
- 差額・％表示：なし
- 保存：原則不要。ただし直近1件（直近の入力状態）を復元できると望ましい（sessionStorage想定）

---

## 3. UI/UX仕様（確定事項）

### 3.1 レイアウト

- スマホ縦画面前提
- 商品カードに A/B/C/D 表示は付けない
- 入力順：容量 → 価格

### 3.2 商品追加/削除

- 初期2件表示
- 「追加」ボタンで1件ずつ追加（最大4件）
- 3件以上のときのみ、各商品に「削除」ボタン表示
- 2件未満にはしない（削除は2件まで）

### 3.3 入力

- 受け付ける値：整数のみ（価格・容量とも）
- 小数が入力された場合：フォーカスアウト時に切り捨てて整数に書き換える
- 0：エラー「数字を入力してください」
- マイナス：エラー「マイナスの値は計算できません」
- 最大値：99999（それ以上は入力不可）
- 数値キーボードが自動で開く設定にする（モバイル）

### 3.4 計算タイミング

- 「表示中の全商品」について価格と容量がすべて有効入力になっている
- その状態で入力欄からフォーカスが外れたら計算する

### 3.5 フォーカス移動

- キーボードの確定（Enter相当）で、次の入力欄へフォーカス移動
- 入力エリア外をタップした場合、フォーカスを外す（どこにもフォーカスが当たらない）

### 3.6 結果/エラー表示

- 未入力がある場合：結果は `--` 表示
- エラーメッセージ：
  - 各入力欄の下に常に表示領域を確保（ガタつき防止）
  - 赤文字、文字サイズ 12px程度
  - 表示タイミングはフォーカスアウト時

### 3.7 追加ボタン位置（暫定）

- 商品リストの最下部に配置
- 実装後に再検討可能

---

## 4. システム構成（全体像）

- 100% フロントエンド（静的サイト）
- 状態はJSメモリ上で管理
- 任意で sessionStorage に直近状態を1件保存
- 外部API / サーバー / DBなし

---

## 5. データモデル（アプリ内状態）

### 5.1 State

- unit: 'g' | 'ml' | 'piece' | 'bag'
- items: Array<Item>（2〜4件）
- Item:
  - qty: number | ''
  - price: number | ''
  - errors:
    - qty: string | ''
    - price: string | ''
- result:
  - unitPrices: number[] | null
  - cheapestIndices: number[]
  - labelType: 'none' | 'cheapest' | 'tie'

### 5.2 保存（任意）

- sessionStorage に直近状態を1件
  - 保存内容：unit、items（qty/price）、表示件数
  - 復元タイミング：初期ロード時

---

## 6. 計算・判定ロジック

### 6.1 単価計算

- base = 100（g/ml） / 1（個/袋）
- unitPrice = (price / qty) \* base
- 表示：小数第1位四捨五入

### 6.2 最安判定

- unitPrices が揃っている時のみ判定
- min = 最小単価
- cheapestIndices = 単価 === min の全index
- 1件のみ → 単独最安
- 2件以上 → 同率最安

---

## 7. バリデーション仕様

### 7.1 共通ルール

- 非数値：エラー
- 0：エラー「数字を入力してください」
- マイナス：エラー「マイナスの値は計算できません」
- 99999超：入力不可

### 7.2 小数

- 入力中は許容
- フォーカスアウト時に切り捨て（Math.floor）
- 切り捨て後 0 なら 0エラー

---

## 8. イベント設計

- unit change：
  - 全商品の容量をクリア
  - 結果を `--` に戻す
- input blur：
  - 小数切り捨て
  - バリデーション
  - 全商品有効なら計算
- Enter：
  - 次入力へフォーカス移動
- outside tap：
  - フォーカス解除
- add：
  - 商品追加（最大4）
- remove：
  - 商品削除（最低2）
